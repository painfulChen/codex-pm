# codex-pm 项目文档（实时版）

> 用途：codex-pm 本身作为“项目管理工具”项目，也需要一份持续演进的文档入口。

## 1. 项目介绍

### 1.1 项目目标

- 多项目统一看板：任务、日志、文档、服务状态一屏管理。
- 机器确认秩序：完成/回滚由机器标识写入，避免“点一下就算完成”的混乱。
- 低成本接入：不依赖数据库与前端构建，HTML 单文件即可用。

### 1.2 当前能力（已落地）

- 任务看板：拖拽排序/流转，支持“申请完成/申请回滚/撤销回滚申请”，机器确认后入日志。
- 日程与版本日志：按日期聚合展示，支持日期条快速跳转。
- 项目文档（可视化）：只读渲染 + 目录 + “圈选建议 -> Todo”。
- 服务状态：按项目聚合 LaunchAgent/端口探测，并支持一键 kickstart。

## 2. 多终端接入（双向读写）

- 每个 Codex 终端可通过 `board_cli.py agent-ping/claim-next/confirm-*` 与看板双向同步。
- 后续可演进为 “board_agent daemon（心跳 + 抢单循环）”，实现终端打开即服从看板。

## 3. 运维入口（实时页）

- 入口：`http://127.0.0.1:8765/pm`
- 数据目录：`~/.clawdbot/pm/`
